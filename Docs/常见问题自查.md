## 安装与启动

### Q: 插件安装后没有出现在任务栏？

**原因**：Native Host 程序未正确启动/未授权。

**解决方法**：

1. 打开 MoeKoeMusic 设置 → 插件管理，确认「任务栏歌词」插件已启用（授权）
2. 检查插件目录是否存在 `MoeKoeTaskbarLyrics.exe`
3. 手动运行 `MoeKoeTaskbarLyrics.exe`，观察是否有错误提示

### Q: 启动时提示缺少 DLL？

**原因**：缺少运行时依赖。

**解决方法**：

- 确认插件目录包含以下文件：
  - `z.dll` (zlib，必须)
  - `kissfft-float.dll` (频谱分析，必须)
  - msvcp140.dll（Visual C++ 2015-2022 Redistributable，非必须）
  - vcruntime140.dll（Visual C++ 2015-2022 Redistributable，非必须）
  - vcruntime140\_1.dll（Visual C++ 2015-2022 Redistributable，非必须）
- 从 [Releases](https://github.com/Yngu196/TaskbarLyrics/releases) 重新下载完整包

### Q: 开机自启动没有生效？

**原因**：自启动注册可能被安全软件拦截。

**解决方法**：

1. 打开插件设置 → 开机自启动，重新启用
2. 检查安全软件是否拦截了注册表/任务计划修改
3. 手动检查生效方式：
   - 注册表: `reg query HKCU\Software\Microsoft\Windows\CurrentVersion\Run`
   - 任务计划: `schtasks /Query /TN MoeKoeTaskbarLyrics_AutoStart`
   - 启动文件夹: `%APPDATA%\Microsoft\Windows\Start Menu\Programs\Startup`

## 歌词显示问题

### Q: 播放音乐后歌词不显示？

**原因**：可能是连接问题或歌词数据未获取。

**解决方法**：

1. 确认 MoeKoeMusic 正在播放且有歌词的歌曲
2. 检查系统托盘图标状态（鼠标停留在托盘图标上）：
   - 正常：显示 `已连接`
   - 断开：显示 `重连中/重连终止/未连接/等待连接`
3. 右键托盘图标，打开设置里的调试日志，然后「重新连接」
4. 查看 `debug.log` 是否有 `[WS] Connected` 日志

### Q: 卡拉OK模式没有卡拉OK效果（没有逐字高亮）？

**原因**：歌曲没有 KRC 格式歌词，只有普通 LRC。

**解决方法**：

- KRC 逐字高亮需要歌曲本身支持 KRC 格式
- LRC 格式歌词只支持整行切换，无法逐字着色
- 这是歌词源的限制，插件本身无法改变

### Q: 双语歌词不显示翻译？

**原因**：歌词文件没有翻译标签，或翻译模式设置错误。

**解决方法**：

1. 确认歌曲歌词是否包含翻译（标准 LRC 格式的同时间戳配对）
2. 右键歌词区域 → 「翻译模式」：
   - 仅原文：不显示翻译
   - 原文+翻译：双语显示（这个只有卡片模式有）
   - 仅翻译：只显示翻译文本

## 连接问题

### Q: 托盘图标灰暗，显示「已断开」？

**原因**：WebSocket 连接失败。

**解决方法**：

1. 确认 MoeKoeMusic 正在运行
2. 打开设置，开启 `调试日志`
3. 右键托盘图标 → 「重新连接」
4. 检查 `debug.log` 中的错误信息(一般在"C:\Users\用户名\AppData\Roaming\moekoemusic\extensions\moeKoe-taskbar-lyrics\debug.log"，你也可以在moekoemusic的插件页面点击“打开插件目录”，然后进入moeKoe-taskbar-lyrics文件夹即可找到日志debug.log)
5. 重启 MoeKoeMusic 和插件

### Q: 连接频繁断开重连？

**原因**：网络不稳定或 MoeKoeMusic 内部通信异常。

**解决方法**：

1. 关闭其他可能占用 WebSocket 端口的程序
2. 检查 `debug.log` 中 `[WS]` 相关日志
3. 尝试重启 MoeKoeMusic

## 任务栏与定位

### Q: 歌词位置不在任务栏上？

**原因**：任务栏检测失败/与桌面美化或任务栏美化工具冲突。

**解决方法**：

1. 确认任务栏正常显示（未自动隐藏）/关闭桌面美化或任务栏美化软件
2. Windows 11 用户：确保任务栏未设置为「自动隐藏」
3. 重启插件，等待任务栏检测完成

### Q: 歌词显示在屏幕外或位置错乱？

**原因**：多显示器或 DPI 缩放异常。

**解决方法**：

1. 右键歌词区域 → 「锁定位置」取消锁定
2. 拖动歌词到正确位置
3. 重新锁定位置
4. 如有多显示器，确保主显示器设置正确

### Q: 歌词被其他窗口遮挡？

**原因**：歌词窗口被非 TOPMOST 窗口覆盖。

**解决方法**：

- 歌词窗口默认是 TOPMOST（置顶），正常情况下不会被遮挡
- 如果被其他置顶窗口遮挡，调整其他窗口设置
- 全屏游戏/视频时，歌词会自动隐藏

## 功能设置

### Q: 设置界面无法打开？

**原因**：D2D 初始化失败，回退到 Win32 对话框。

**解决方法**：

1. 确认显卡驱动正常
2. 尝试右键托盘图标 → 「设置...」
3. 检查 `debug.log` 是否有 D2D 相关错误

### Q: 字体/颜色设置修改后没有变化？

**原因**：配置未保存或渲染未更新。

**解决方法**：

1. 在设置界面点击「保存」按钮
2. 确认配置文件 `config.json` 已更新（检查 `debug.log`）
3. 重新打开设置界面确认修改生效

### Q: 全屏时歌词不自动隐藏？

**原因**：全屏检测功能未开启。

**解决方法**：

1. 打开设置 → 开启 调试日志
2. 重新全屏
3. 检查 `debug.log`

## 系统兼容性

### Q: Windows 11 上歌词显示异常？

**原因**：Windows 11 任务栏布局变化。

**解决方法**：

- 本插件已针对 Windows 11 优化
- 确保使用最新版本
- 如有问题，尝试：
  1. 重启插件
  2. 检查任务栏是否为默认设置（居中/左对齐均可）
  3. 报告 Issue 并附上系统版本和日志

### Q: 任务栏在屏幕顶部/左侧/右侧时歌词位置不对？

**原因**：任务栏方位检测延迟。

**解决方法**：

- 移动任务栏位置后，插件会在几秒内自动调整
- 如长时间未调整，右键托盘图标 → 「重新连接」触发重新检测
- 如果未得到解决，报告 Issue 并附上系统版本和日志

## 日志与调试

### Q: 如何查看日志？

**方法**：

- 日志文件：`debug.log`，位于 `MoeKoeTaskbarLyrics.exe` 同目录(一般在"C:\Users\用户名\AppData\Roaming\moekoemusic\extensions\moeKoe-taskbar-lyrics\debug.log"，你也可以在moekoemusic的插件页面点击“打开插件目录”，然后进入moeKoe-taskbar-lyrics文件夹即可找到日志debug.log)
- 日志包含连接状态、错误信息、配置变更等
- 提交 Issue 时请附上相关日志片段（删除敏感信息）

### Q: 如何启用更详细的日志？

**方法**：

1. 打开设置
2. 启用「调试日志」
3. 日志会输出更多内部状态信息，有助于排查问题

### Q: 日志文件太大怎么办？

**解决方法**：

- 日志文件会在每次启动时追加，不会自动清理
- 可以手动删除旧日志
- 日志对排查问题很重要，建议保留最近几天的内容

## 其他问题

### Q: 播放控制按钮不响应点击？

**原因**：按钮功能依赖 MoeKoeMusic 通信。

**解决方法**：

1. 确认 WebSocket 连接正常
2. 确认歌词区域未开启「完全锁定」
3. 检查 `debug.log` 是否有按钮点击日志

### Q: 如何反馈新问题？

**方法**：

1. 在 [Issues](https://github.com/Yngu196/TaskbarLyrics/issues) 提交 Bug 报告
2. 使用错误报告模板，填写：
   - 操作系统版本
   - 任务栏位置
   - 插件版本
   - `debug.log` 相关片段
3. 如有截图/录屏请一并附上
