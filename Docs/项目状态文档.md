## 3. 文档规划 vs 实际实现 — 关键差异

| #  | 文档规划                   | 实际实现                   | 原因                          |
| -- | ---------------------- | ---------------------- | --------------------------- |
| 1  | SetParent 嵌入子窗口        | 独立浮动窗口覆盖               | Win11 任务栏结构变化，SetParent 不稳定 |
| 2  | C++20                  | C++17                  | 功能无影响                       |
| 3  | SetDrawingEffect 着色    | PushAxisAlignedClip 裁剪 | 更简单可靠的实现                    |
| 4  | 绑定模式+进程监控              | Native Host 托管（v0.4.0） | 更优的生命周期管理                   |
| 5  | WS\_EX\_TRANSPARENT 穿透 | 不使用穿透                  | 需接收鼠标事件支持交互                 |
| 6  | 日志写入 %TEMP%            | exe 同级 debug.log       | 可移植路径                       |
| 7  | 手动编辑 JSON              | D2D 原生 GUI             | 完整配置界面                      |
| 8  | 无拖动功能                  | 支持拖动定位                 | 新增需求                        |
| 9  | 无 Z-order 保持           | 三重 TOPMOST 防护          | 修复点击被覆盖问题                   |
| 10 | TerminateThread 强杀     | 协作式 join() 退出          | 代码审查安全隐患修复                  |
| 11 | 散落魔数                   | constants.h 集中管理       | 代码审查建议                      |
| 12 | 无异常恢复                  | WM\_TIMER catch 重试     | 防止渲染器卡死                     |
| 13 | 无配置校验                  | std::clamp 范围验证        | 防止非法值                       |
| 14 | 无长歌词滚动                 | 跑马灯 6 状态机              | 设计需求                        |
| 15 | 无 API 自动开启             | api\_enabler 模块        | 用户需求                        |
