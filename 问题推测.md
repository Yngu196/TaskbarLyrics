怀疑的问题 1

任务栏里存在：

Windows.UI.Composition.DesktopWindowContentBridge

这是 Win11 新任务栏的核心绘制层。

而代码里：

SetParent(hwnd_, hTaskbar_);

把歌词窗口挂到了：

Shell_TrayWnd

下面。

但 Win11 的任务栏内容实际上是：

Shell_TrayWnd
 └─ DesktopWindowContentBridge
      └─ 所有可见UI

歌词窗口虽然存在：

Shell_TrayWnd
 ├─ MoeKoeTaskbarLyricsClass
 └─ DesktopWindowContentBridge

但被 DesktopWindowContentBridge 完全盖住了。

验证方法

临时改：

SetParent(hwnd_, nullptr);

不要嵌入任务栏。

然后：

SetWindowPos(
    hwnd_,
    HWND_TOPMOST,
    100,
    100,
    600,
    40,
    SWP_SHOWWINDOW);

如果能看到歌词：

梦见你 (《时间之子》电影主题曲/片尾曲) - 周深

说明渲染完全正常。

问题就是任务栏层级。

怀疑的问题 2

这里：

const DWORD exStyle =
    WS_EX_TRANSPARENT |
    WS_EX_NOACTIVATE |
    WS_EX_TOOLWINDOW;

没有：

WS_EX_LAYERED

如果 Renderer 使用：

Clear(0,0,0,0)

透明背景，

而窗口又不是 Layered，

有些系统上会变成：

完全透明

导致文字也不可见。

怀疑的问题 3（概率最大）

看到：

child[4]
Windows.UI.Composition.DesktopWindowContentBridge
rect=(0,0,1920,48)

它占满了整个任务栏：

0 → 1920

而代码：

if (cw < tbWidth - 10)
{
    taskListRect = ci.rect;
    foundTaskList = true;
}

结果：

foundTaskList = false

日志也证明：

foundTaskList=0

因此歌词窗口被放到：

1004 ~ 1652

这个区域。

但这整个区域其实仍然属于：

DesktopWindowContentBridge

覆盖层。

建议立刻做的调试

在 Create 后加：

::MessageBoxW(
    nullptr,
    L"Taskbar window created",
    L"DEBUG",
    MB_OK);

然后：

SetParent(hwnd_, nullptr);

再运行。

如果歌词出现：

说明：

歌词系统正常
任务栏嵌入失败
Win11 更可靠的方案

不要：

SetParent(hwnd_, Shell_TrayWnd);

改成：

WS_POPUP

独立窗口。

然后每帧：

GetWindowRect(taskbar);

跟随任务栏位置。

效果：

┌────────────────────┐
│      任务栏歌词     │
└────────────────────┘

覆盖在任务栏上方。

类似：

TrafficMonitor
ExplorerBlurMica
TranslucentTB

这些 Win11 工具的实现方式。

兼容性会比嵌入 Shell_TrayWnd 高很多。
